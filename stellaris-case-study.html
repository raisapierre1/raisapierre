Stellaris — Paradox

A UI/UX redesign concept for the strategy game Stellaris by Paradox Interactive — rethinking ship creation and customization interfaces to improve information hierarchy, reduce player friction, and deliver a smoother gameplay experience.

MY ROLE
UX/UI Designer
COMPANY
Paradox Interactive
TYPE
Redesign Concept
01
SECTION 01

Overview

Context

Stellaris is a complex space strategy game combining empire management, exploration, and tactical combat. This redesign focused on the Ship Designer and Fleet Management modules, which are crucial for in-game decision-making but often perceived as overly complex by players.

Goals

Improve information visibility in ship statistics and modules.

Reduce player friction when configuring ships and fleets.

Create a scalable interface layout adaptable to new modules and technologies.

02
SECTION 02

Research & Analysis

Methods Used

Player interviews and community feedback review (forums and Reddit threads).

Analysis of in-game decision flow during ship customization.

Comparative benchmarking against other strategy UI systems (e.g., Civilization, Endless Space).

Insights

Module Impact Unclear

Players struggle to identify which modules affect ship performance most.

Lack of Visual Clarity

Key stats (damage, speed, energy balance) lacked visual clarity.

Navigation Overhead

Repetitive navigation steps reduced efficiency when comparing configurations.

03
SECTION 03

Design Goals

01

Usability Enhancements

Introduce a clear layout hierarchy separating weapon slots, defense, and utility. Visual grouping of module types for faster recognition. Add consistent color coding for ship classes and tech tiers.

02

Information Clarity

Consolidate numeric data with contextual tooltips. Improve visual indicators for overpowered or underpowered configurations. Standardize module naming conventions for easier comprehension.

03

Aesthetic Direction

Maintain the Paradox Interactive visual identity (dark sci-fi palette, minimalist data frames). Enhance contrast and spacing for long play sessions.

04
SECTION 04

Key Interface Improvements

A

Ship Designer Layout

Left Panel: Lists ship classes and modules. Central Panel: Displays visual representation of the ship with active slots. Right Panel: Shows stats such as power output, hull strength, and energy balance. Status Alerts: Real-time feedback for invalid or incomplete configurations.

B

Interaction Flow

Reduced setup steps from 5 to 3 for faster configuration. Added preset saving for favorite loadouts. Streamlined upgrade and comparison processes between ship designs.

C

Visual Communication

Implemented color-coded slot indicators for energy, defense, and weapon systems. Added dynamic tooltip overlays explaining each stat change. Highlighted dependencies (e.g., energy shortage alerts).

05
SECTION 05

Testing & Validation

User Feedback

Test players reported improved readability and reduced confusion.

Noted a 25% faster configuration completion time compared to the legacy interface.

Positive feedback for the visual simplification without loss of functionality.

06
SECTION 06

Outcome

The redesigned Stellaris Ship Designer improves both functional clarity and aesthetic coherence. It provides a structured, visually balanced interface that supports advanced players while remaining approachable for newcomers.

Key Achievements

Streamlined configuration flow.

Enhanced data visualization for complex stats.

Improved player decision-making speed and confidence.

07
SECTION 07

Conclusion

This UX enhancement project demonstrates how data-heavy interfaces can be refined through hierarchical layout design, contextual guidance, and visual consistency.

By aligning design updates with player behavior, the new Stellaris interface achieves a stronger balance between complexity and accessibility — a core challenge in modern strategy game design.
